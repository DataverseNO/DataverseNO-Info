# Collection Management Guidelines

This guide is for collection managers at DataverseNO partner institutions. Collection managers are responsible for administering their institution's collection and granting depositor access.

## User and access administration

### Creating user accounts

Users from Norwegian research institutions with Feide credentials get automatic access to their institution's DataverseNO collection on first login. No manual account creation is needed.

Users logging in via ORCID must verify their institutional email before access is granted.

All other users (without Feide or institutional ORCID) must apply by contacting [support@dataverse.no](mailto:support@dataverse.no).

### Granting deposit access

To grant a user deposit access to your collection:

1. Navigate to your collection in DataverseNO
2. Go to **Edit → Groups**
3. Find the user and add them to the **Dataset Creator** group

For complex access arrangements or creating subcollections and special collections, contact [support@dataverse.no](mailto:support@dataverse.no).

### Edit access to individual datasets

Collection managers can grant other users edit access to specific datasets directly from the dataset's **Edit → Permissions** menu.

## Collection types

DataverseNO supports three collection types:

| Type | Description |
|---|---|
| Institution collection | Main collection for a partner institution |
| Subcollection | Thematic or departmental sub-division of an institution collection |
| Special collection | Collections with non-standard access or licensing arrangements |

Contact [support@dataverse.no](mailto:support@dataverse.no) to create subcollections or special collections.

## Metadata harvesting

OAI-PMH harvesting sets exist for all institutional collections. Harvesting addresses are listed on the [About DataverseNO](../../about/index.md) page.

## Support

For questions about collection administration, contact [support@dataverse.no](mailto:support@dataverse.no).
