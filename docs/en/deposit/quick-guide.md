# Quick guide & checklist

Follow the steps (1-3) to make sure your dataset complies with the DataverseNO deposit guidelines.

## 1. Prepare your data

- ❑ **File naming and organization:**
    - Follow good practice for file naming and organization.
    - Follow general recommendations for spreadsheets/tabular files.
- ❑ **Convert to preferred file formats:**
    - Choose a preferred file format that is suitable for long term-preservation.
    - Convert your data files into a preferred file format before archiving.
- ❑ **Prepare a ReadMe file:**
    - Describe your data in a ReadMe file, using a template.
    - Save in plain text format (.txt) with Unicode UTF-8 character encoding.
- ❑ **File and dataset size:**
    - The size of an individual file should not exceed 100 GB.
    - The size of a total dataset should not exceed 200 GB.
    - For larger files or datasets, contact our support service.

## 2. Deposit your data

- ❑ **Create an account**
- ❑ **Create a draft:**
    - Choose your home institution or project from the top banner.
    - Click the 'Add Data' button and select 'New Dataset'.
- ❑ **Enter metadata:**
    - Enter as much metadata as possible.
    - Check out the section on metadata in the DataverseNO deposit guidelines.
    - Remember to click 'Save Dataset' at the bottom of the page.
- ❑ **Enter more metadata:**
    - Select the 'Metadata' tab and click the 'Add + Edit Metadata' button.
    - (Note that more metadata fields will appear)
    - Click 'Save Dataset' at the bottom of the page.
- ❑ **Confirm or specify a license**
    - Standard license for reuse is CC0. No suitable license? Contact our support service.
- ❑ **Upload files**
    - A dataset must not contain more than 300 files. More files? See deposit guide.
    - A file upload should not exceed 200 GB in total size.
    - Select the 'Files' tab and click 'Upload Files'.
    - Upload single files: click 'Select Files' to 'Add' button.
    - Want to keep folder structure? Click the 'Upload a Folder' button.
    - Click 'Save' button when done.
- ❑ **Share your data before publishing**
    - Need to share a dataset before publication, for instance with collaborators or peer-reviewers? Contact your local support service.
    - You may apply an embargo to files in a published dataset for a limited time period.
- ❑ **Submit your data for review (curation)**
    - Prepared your data according to the guidelines? Click 'Submit for Review.'
    - Note that your dataset is only a draft. The dataset DOI is not active yet.
    - A curator from your institutional collection will review your dataset for compliance with the deposit guide and, if necessary, inform you about changes that need to be made before publication. Allow a few days for this process. You will receive a curation report by email, and the dataset will be returned to you for revision.
    - If needed, make the recommended changes. Alternatively, explain why you don't agree with the recommendations.
    - Done with the revision? Click the 'Submit for Review' button.
- ❑ **Get your data published**
    - Once the curator has approved the dataset and you have confirmed it is ready for publication, the curator will publish it. The dataset DOI is now activated.

## 3. Refer to your dataset

- ❑ **Refer to your published dataset:**
    - Use the reference in the blue box provided by the repository. The box is at the top of the dataset page that appears when clicking the DOI link.

NOTE: It is possible to modify your dataset after publication. This will create a new version of the dataset that also needs to be reviewed. Previous versions are still accessible.
