# Return dataset to author

If the dataset requires changes before publication, return it to the depositor with clear feedback.

## When to return

Return the dataset when:

- Required metadata fields are missing or incorrect
- The README is absent or insufficient
- Files are missing, corrupted, or in unacceptable formats
- Sensitive data is present
- The license is inappropriate

## How to return

1. In DataverseNO, click **Return to Author**
2. Write a message explaining what needs to be changed — be specific and constructive
3. The depositor will receive a notification and can make corrections

## Writing feedback

Be specific. Instead of:

> "The metadata needs improvement."

Write:

> "Please add a description that explains what the data files contain, how the data was collected, and what the column names in `data.csv` mean."

The depositor may not be familiar with curation standards — explain *why* the change is needed where it helps.
