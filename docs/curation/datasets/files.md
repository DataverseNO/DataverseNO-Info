# Files

Review the uploaded files for completeness, format, and documentation quality.

## Checklist

- [ ] A README file is present and informative
- [ ] File formats are open or accompanied by open equivalents
- [ ] File names are descriptive and consistent
- [ ] No personal or sensitive data present
- [ ] Data files can be opened and are not corrupted

## README quality

The README should explain:

- What each file contains
- How the data was collected or generated
- Variable names and units
- Any software needed to work with the files

A README that says only "data from the project" is not sufficient.

## File formats

Prefer open formats. If a depositor has only submitted proprietary formats (e.g. `.xlsx`, `.sav`), request that open equivalents are added (`.csv`, `.tsv`).

## Missing files

If files referenced in the README or description are missing from the deposit, return the dataset to the author.
