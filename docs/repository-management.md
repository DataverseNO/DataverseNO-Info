# Repository management

This guide is for DataverseNO repository administrators at UiT The Arctic University of Norway.

## User and access administration

### Creating user accounts

Users from Norwegian research institutions with Feide receive automatic access on first login. Individual researchers without an institutional partner agreement can deposit up to 10 GB at no cost — they apply via the form at [dataverse.no](https://dataverse.no).

### When an institution becomes a partner

When a new institution joins as a DataverseNO partner, two tasks are required:

1. **Transfer collection access** — update group memberships so existing users are placed under the new institutional collection
2. **Move existing datasets** — relocate any datasets the institution's researchers deposited in the root DataverseNO collection into the new institutional collection

Contact [support@dataverse.no](mailto:support@dataverse.no) to coordinate this process.

## OAI-PMH harvesting

OAI-PMH harvesting sets are maintained for all institutional collections and for the full DataverseNO repository. Set addresses are listed on the [About DataverseNO](about/index.md) page.

When a new institutional collection is created, update the OAI-PMH set list accordingly.

## re3data registry

DataverseNO is registered in [re3data.org](https://re3data.org). Keep the re3data entry up to date when significant changes occur (new partners, policy updates, contact changes).

## Large file uploads

Deposits exceeding 200 GB require prior coordination with the UiT infrastructure team. Refer depositors to [support@dataverse.no](mailto:support@dataverse.no) before they attempt large uploads.

## Long-term preservation

Tasks in connection with long-term preservation follow the [Preservation Policy](about/index.md). Accessibility statements must be updated annually.

## Support

For questions about repository administration, contact [support@dataverse.no](mailto:support@dataverse.no).
