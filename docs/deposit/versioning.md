# Versioning

DataverseNO supports modifying datasets after publication. All previous versions remain archived and publicly accessible.

## Modifying a published dataset

You can update metadata and/or data files at any time after publication. Each update creates a new dataset version, which must be approved by a curator from your institutional collection before it goes live.

Remember to **Submit for Review** after making changes — saving a draft does not notify the curator.

## Version types

| Type | When it applies |
|---|---|
| Minor release | Metadata-only changes (title, description, keywords, etc.) |
| Major release | Data files added, removed, or changed |

## Version history

All versions of a dataset are preserved and remain publicly accessible. Older versions can be accessed from the **Versions** tab on the dataset landing page. Each version is independently citable.

## Deaccession

Published datasets cannot be deleted. In exceptional circumstances, deaccession is possible — the data files become restricted, but the descriptive metadata remains visible so that citations remain resolvable.

Deaccession requires approval. Contact your institution's DataverseNO support to request it.
